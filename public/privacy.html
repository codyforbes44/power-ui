01 Overview

Async is built on a local-first architecture. This is not a marketing phrase — it's a hard constraint baked into how the application works. There is no server owned by this project that your conversations pass through, no database that stores your messages, and no account system that ties your activity to an identity.

When you use Async, every conversation you have, every setting you configure, and every API key you add is held entirely on your device, inside your browser's own storage. The application is a static front-end that talks directly to the AI provider APIs you choose to use — nothing sits in between.

Architecture in one sentence Your browser ↔ AI provider API. That's it. No Async servers in that chain.

This policy explains exactly what is stored, where, and how it is protected — so you can make an informed decision about whether this tool is right for you.

02 Information We Do Not Collect

The list of things we do not collect is more useful than a list of things we do. To be explicit:

  • No telemetry. We don't track which features you use, how often you open the app, or how many messages you send.
  • No analytics. There is no Google Analytics, Mixpanel, Amplitude, or any similar service embedded in this application.
  • No crash reports. If the app crashes, we don't receive a report. You'd have to tell us via GitHub.
  • No usage metrics. Token counts, message lengths, conversation durations — none of it is sent anywhere.
  • No IP logging. We don't run the servers, so we can't log your IP. The AI providers receive your IP when you call their APIs — see Section 5 for details.
  • No advertising IDs or fingerprinting. We make no attempt to identify you across sessions, devices, or sites.
  • No account data. There is no sign-up form, no email field, no password we store.

The simplest mental model: if we don't run a server that receives your data, we can't collect your data. That is the actual state of affairs.

03 Information Stored Locally on Your Device

Async does store data — just entirely on your device, in your browser's own storage APIs. Here's a full breakdown of what gets saved and why:

localStorage — persisted across browser sessions
  • Conversation history — your messages and AI responses, stored so the app can restore them when you reopen it.
  • App settings — theme, model selection, temperature, system prompt, sidebar state, etc.
  • Encrypted API key vault — your API keys stored in AES-256-GCM encrypted form. The decryption key is never stored here; see Section 4.
  • Memory entries — any facts or context you've explicitly asked the app to remember across conversations.
  • Conversation metadata — titles, creation dates, and model used, so the sidebar list can be rendered without decrypting message content.
sessionStorage — cleared when you close the tab
  • Vault decryption key — the key used to decrypt your API key vault for the current session. It's held here specifically because sessionStorage is wiped when the tab closes, limiting the window of exposure.

You can clear all of this at any time by opening your browser's developer tools and clearing site data for the origin, or by using the built-in "Clear All Data" option in the app's settings panel.

None of this data ever leaves your device unless you explicitly export it using the app's export feature.

04 API Key Security

Your API keys are the most sensitive thing the application touches. We treat them accordingly. Here is exactly how they are protected:

Encryption pipeline
  • Key derivation: When you set a vault password, PBKDF2-SHA-256 is used with 310,000 iterations and a random 16-byte salt to derive a 256-bit encryption key. This makes brute-force attacks computationally expensive.
  • Encryption: The derived key encrypts your API keys using AES-256-GCM. Each encryption operation uses a freshly generated random 96-bit initialization vector (IV), making ciphertexts unique even for identical inputs.
  • Storage: Only the encrypted ciphertext, the salt, and the IV are written to localStorage. The raw API key is never stored.
  • Session key: Once you unlock the vault with your password, the derived encryption key lives only in sessionStorage. It's gone the moment you close the tab.

All cryptographic operations are performed by the browser's native Web Crypto API (window.crypto.subtle). No third-party crypto library is involved.

If you forget your vault password, your API keys cannot be recovered — this is intentional. There is no password-reset mechanism because there is no server to send a reset email from. Simply clear the vault and re-enter your keys.

05 Third-Party Services

When you send a message, it goes directly from your browser to the API endpoint of the AI provider you have configured — Anthropic, OpenAI, Google, Mistral, or any other compatible provider. We are not involved in that request.

This means those providers receive:

  • Your messages and any attached files you send
  • Your IP address (standard for any HTTP request)
  • Your API key (sent in the Authorization header, which is why we encrypt them at rest)
  • Any metadata the API requires (model name, parameters, etc.)

Each provider has their own privacy policy governing how they handle this data. You should read them:

  • Anthropic Privacy Policy
  • OpenAI Privacy Policy
  • Google Privacy Policy
  • Mistral AI Privacy Policy

Google Fonts is also loaded by this page to render the Inter and Outfit typefaces. When your browser fetches font files from Google's CDN, Google receives your IP address. See Google's privacy policy for how they handle font CDN requests.

Aside from the AI providers you choose and Google Fonts, no other third-party services are loaded by this application.

06 Server Mode (Optional)

Async includes an optional server.py that you can run locally to enable a small number of server-assisted features — primarily real-time updates via Server-Sent Events (SSE) for multi-tab synchronisation, and local file-based storage as an alternative to the browser's localStorage.

Key things to understand about server mode:

  • It runs on your machine. You start it yourself with python server.py. It binds to localhost by default and is not exposed to the internet unless you explicitly configure it to be.
  • Data stays local. Any data the server persists is written to a /data directory next to server.py on your own filesystem.
  • No external transmission. The server does not send data anywhere. It receives API requests from your browser and stores or broadcasts them locally.
  • SSE broadcast. If you have multiple tabs open, the server can push updates between them over SSE. This traffic never leaves localhost.
  • Your responsibility. If you choose to expose the server on your network (e.g., via port forwarding), you are responsible for securing that access. The server does not include authentication by default.

If you never run server.py, the application operates entirely in browser-only mode and server mode is simply irrelevant.

07 Cookies & Storage

Async does not set any cookies — first-party or third-party. All persistent state is managed through the browser's Web Storage APIs:

  • localStorage — for data that should persist between browser sessions (conversations, settings, encrypted API keys).
  • sessionStorage — for data that should be cleared when you close the tab (the vault decryption key).

No tracking cookies, no session cookies, no advertising cookies. The only third-party storage that might be set is by Google Fonts if your browser caches font files under Google's domain — that is outside our control and governed by Google's policies.

08 Children's Privacy

Async is not directed at children under the age of 13, and we do not knowingly collect personal information from children under 13. Since the application requires users to obtain and manage their own API keys from AI providers — each of which has their own age requirements — the practical minimum age for using this tool is determined by those providers' terms of service.

Anthropic, OpenAI, and other major providers generally require users to be at least 13 years old, with parental consent often required for users under 18. Please review the terms of service of whichever provider you use.

09 Changes to This Policy

If this privacy policy changes in a meaningful way, we will:

  • Update the "Last updated" date at the top of this page
  • Create a GitHub release note or commit describing what changed and why
  • For significant changes, open a GitHub Discussion to allow community feedback before the change takes effect

Because there is no account system, we cannot email you about policy changes. If you want to stay informed, watch the repository on GitHub to receive notifications about releases and significant updates.

Continued use of the application after a policy change is published constitutes acceptance of the updated policy. If you disagree with a change, you can stop using the application and clear your local data at any time.

10 Contact

If you have questions about this privacy policy, believe there is a privacy issue with the application, or want to report a security concern related to data handling, please open an issue on GitHub:

GitHub Issues github.com/codyforbes/async-ai/issues — tag your issue with the privacy or security label.

For sensitive security disclosures, please use GitHub's private Security Advisory feature rather than opening a public issue. This allows us to review and address the concern before it is publicly visible.

We are a small open-source project and do not have a dedicated privacy team or a formal data protection officer. Response times may vary, but we take privacy concerns seriously and will respond to every legitimate inquiry.
