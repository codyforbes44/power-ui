Getting Started

Quick Start

Async is a zero-dependency, browser-native AI chat interface. No build step, no Node.js, no npm. Just open a file and start chatting.

✨
Fastest path to running Download or clone the repo, then open app/index.html in any modern browser. That's it.
1

Download or clone the repository

Get the source from GitHub — either download a ZIP or use git clone.

2

Open app/index.html in your browser

Double-click the file or drag it onto a browser window. No local server required.

3

Enter an API key

On first launch, the settings panel opens automatically. Paste your key and select a provider. Keys are encrypted in localStorage using AES-GCM.

4

Start a conversation

Type in the composer and press Enter (or Shift+Enter for a newline). Streaming responses appear in real time.

Shell 
# Clone the repository
git clone https://github.com/your-org/async-ai.git
cd async-ai

# Open directly in browser (macOS)
open app/index.html

# Open directly in browser (Linux)
xdg-open app/index.html

# Open directly in browser (Windows)
start app/index.html
ℹ️
Want multi-device sync? Run python3 server.py to enable real-time sync across devices via SSE. See Server Mode for details.
Prerequisites

Requirements

Async runs entirely in the browser. The only hard requirement is a modern browser and at least one AI provider API key.

Because there is no build toolchain, runtime dependencies are minimal. The app uses standard browser APIs — fetch, localStorage, SubtleCrypto, EventSource — that have been available for years.

Requirement Details Status
Chrome / Edge Version 100 or later Supported
Firefox Version 110 or later Supported
Safari Version 16 or later (macOS & iOS) Supported
AI Provider API Key Anthropic, OpenAI, Google, or Groq Required
Node.js / npm Not required for basic use Optional
Python 3.8+ Only needed for Server Mode Optional
⚠️
File:// protocol limitations When opening via file://, some browsers restrict certain APIs. If you encounter issues with clipboard or crypto, use Server Mode (python3 server.py) which serves the app over http://localhost:8080 where all APIs work reliably.
Setup

Installation

Choose the method that fits your workflow. Option A is the simplest; Option B unlocks multi-device sync and a shared conversation history.

Option A

Direct File Access

Open app/index.html straight from your filesystem. No server, no dependencies. Data persists in browser localStorage.

Option B

Python Dev Server

Run python3 server.py to serve the app at localhost:8080. Enables real-time multi-device sync, conversation sharing, and admin controls.

Option A — Direct File

Simply open the file. All conversations, settings, and memories are stored in your browser's localStorage and persist between sessions automatically.

Shell 
# Navigate to the project folder
cd async-ai

# Open in default browser
open app/index.html   # macOS
start app/index.html  # Windows

Option B — Python Server

The included server.py is a lightweight HTTP + SSE server. It requires only the Python standard library — no pip install needed.

Shell 
# Start the server (Python 3.8+ required)
python3 server.py

# The app is now available at:
# http://localhost:8080/app/index.html

# Optional: specify a custom port
python3 server.py --port 9000
💡
Network access By default the server binds to 0.0.0.0, making it accessible on your local network. Other devices (phone, tablet, second laptop) can connect using your machine's local IP, e.g. http://192.168.1.42:8080.

Adding a key

  1. Click the Settings (⚙) icon in the sidebar, or press Ctrl+,.
  2. Select the API Keys tab.
  3. Choose your provider from the dropdown.
  4. Paste the key and click Save. The field will blur and display only the last 4 characters.

You can store keys for multiple providers simultaneously and switch between them per-conversation in the model selector.

How keys are stored

Keys are encrypted using the Web Crypto API (SubtleCrypto.encrypt) with AES-GCM 256-bit encryption before being written to localStorage. A random 96-bit IV is generated for each write. The encryption key itself is derived from a browser-fingerprint seed unique to your device.

JavaScript — Simplified encryption flow 
// Key derivation (simplified)
const rawKey = await crypto.subtle.importKey(
  'raw', deviceSeed, { name: 'AES-GCM' }, false, ['encrypt', 'decrypt']
);

// Encryption with random IV
const iv = crypto.getRandomValues(new Uint8Array(12));
const ciphertext = await crypto.subtle.encrypt(
  { name: 'AES-GCM', iv }, rawKey, encode(apiKey)
);

// Stored as base64(iv + ciphertext)
localStorage.setItem('enc_key_anthropic', toBase64(iv, ciphertext));
🔒
Keys are device-local Encrypted keys cannot be transferred between browsers or devices. If you use Server Mode, you will need to re-enter your key in each browser instance you connect with. Keys are never transmitted to the sync server.
Features

Skills

Skills are reusable instruction sets — like specialized system prompts — that shape how the AI responds for a specific task or domain.

Think of a Skill as a pre-configured expert mode. You might have a Code Review skill that instructs the model to focus on security and performance, or a Marketing Copy skill that emphasizes persuasion and brand voice.

Skills are stored as plain JSON in localStorage (or synced via the server in Server Mode) and can be created, edited, duplicated, and deleted from the Skills Library panel.

Auto-detection

Async can automatically detect which skill to apply based on the content of your first message. Auto-detection uses a lightweight keyword and pattern matching engine:

  • Messages containing code fences (`` ``` ``) → suggests Code skills
  • Messages mentioning "translate", "Spanish", "French", etc. → suggests Translation skills
  • Messages with "write a blog", "draft an email" → suggests Writing skills

A toast notification appears with the detected skill. Click Apply to accept, or dismiss to continue without it.

Manually applying a skill

  1. Open the Skills panel (the ⚡ icon in the toolbar above the composer).
  2. Browse or search the library by name or tag.
  3. Click a skill card to preview its system prompt.
  4. Click Apply to Conversation. The skill's prompt is injected as the first system message.
JSON — Skill structure 
{
  "id": "skill_code_review",
  "name": "Code Review",
  "description": "Senior engineer focused on security and performance",
  "tags": ["code", "engineering", "review"],
  "system_prompt": "You are a senior software engineer reviewing code for\nsecurity vulnerabilities, performance bottlenecks, and\nmaintainability. Always explain your reasoning...",
  "model": "claude-3-5-sonnet-20241022",
  "temperature": 0.3,
  "workspace_id": "ws_default"
}
💡
Skills are workspace-scoped Skills created in one workspace are not visible in others unless you explicitly share them. This lets you keep client-specific or project-specific skills neatly isolated.
Features

Memory

Memory lets the AI remember facts, preferences, and context across conversations — so you never have to repeat yourself.

Memories are short, factual statements stored in a searchable index. When you start a new conversation, Async performs a semantic similarity search and injects the most relevant memories as context before your first message.

Adding memories

There are two ways to add a memory:

  • Manual entry — Open the Memory panel (🧠 icon), click + Add Memory, type a fact, and hit Save.
  • Auto-extract — At the end of a conversation, click Extract Memories. The AI reads the conversation and generates a set of concise factual statements for your review. Accept, edit, or discard each one.

Searching memories

The Memory panel includes a live search bar. Type any keyword or phrase and the list filters instantly. Results are ranked by recency and relevance. You can also filter by workspace using the dropdown at the top of the panel.

Workspace scope

Every memory belongs to a workspace. Memories in Workspace A are never injected into conversations in Workspace B. There is also a Global memory scope for facts you want available everywhere (e.g., your name, timezone, preferred language).

ℹ️
Token budget By default, up to 8 memories are injected per conversation to keep token usage manageable. You can adjust this limit in Settings → Memory → Max memories per context.
Features

Workspaces

Workspaces are isolated environments — each with its own conversations, skills, memories, and system prompt. Perfect for separating work, personal, and client projects.

When you first launch Async, a Default workspace is created automatically. Everything you do lives inside a workspace. Switching workspaces is instant and data never bleeds between them.

Creating a workspace

  1. Click the workspace name in the top-left corner of the sidebar.
  2. Select + New Workspace from the dropdown.
  3. Enter a name and (optionally) a default system prompt, then click Create.

Per-workspace system prompts

Each workspace can have a default system prompt that is prepended to every new conversation started in that workspace. This is separate from (and combined with) any skill you apply. A common pattern:

  • Workspace prompt: Sets the tone and ground rules ("You are a helpful assistant for Acme Corp. Be concise. Never mention competitors.").
  • Skill: Adds domain expertise on top ("You are a SQL expert…").

Isolation

The following are fully isolated per workspace:

  • Conversation history
  • Skills library
  • Memory bank
  • Default model and temperature settings
  • System prompt

API keys are not isolated — they are shared across all workspaces by design, since keys belong to you, not to a project.

💡
Keyboard shortcut Press Ctrl+Shift+W (or Cmd+Shift+W on macOS) to open the workspace switcher instantly.
Advanced

Server Mode

Run a lightweight Python server to unlock multi-device sync, shared conversation history, and real-time collaboration via Server-Sent Events.

The included server.py uses only the Python standard library (http.server, json, threading). It handles static file serving, a JSON REST API for data persistence, and an SSE stream for push updates to all connected clients.

Starting the server

Shell 
# Default: binds to 0.0.0.0:8080
python3 server.py

# Custom port
python3 server.py --port 9090

# Bind to localhost only (disable LAN access)
python3 server.py --host 127.0.0.1

# With verbose logging
python3 server.py --verbose

Connecting devices

Once the server is running, any device on the same network can connect:

  1. Find your machine's local IP address (ifconfig / ipconfig).
  2. On another device, navigate to http://<your-ip>:8080/app/index.html.
  3. The app automatically detects the server and switches from localStorage to server-backed sync.

All conversations, workspaces, memories, and skills are stored in data/ as flat JSON files. No database required.

How SSE sync works

Clients subscribe to /api/events using the browser's native EventSource API. When any client makes a change (new message, renamed conversation, etc.), the server broadcasts a typed event to all other subscribers. Changes appear in real time without polling.

Server-Sent Events — Event types 
// Events broadcast by the server
event: conversation.created   // New conversation started
event: conversation.updated   // Message added or edited
event: conversation.deleted   // Conversation removed
event: memory.updated         // Memory bank changed
event: skill.updated          // Skill created or modified
event: workspace.switched     // Active workspace changed
event: heartbeat               // Keep-alive every 30s
⚠️
Security note Server Mode has no authentication by default when accessed on localhost. If you expose it on a LAN or public network, enable admin authentication in Settings and consider placing it behind a reverse proxy with TLS. See Admin Dashboard for details.
Advanced

Admin Dashboard

A password-protected control panel for managing users, reviewing usage stats, purging data, and controlling server-mode settings.

The Admin Dashboard is accessible at /app/admin.html when the server is running. It provides a bird's-eye view of all activity on the instance.

Accessing the dashboard

1

Start the server

Run python3 server.py and confirm it is listening on port 8080.

2

Navigate to admin

Open http://localhost:8080/app/admin.html in your browser.

3

Login with default credentials

Username: admin — Password: admin123. Change this immediately on first login.

4

Change your password

Go to Settings → Admin → Change Password. The new hash is stored in data/admin.json.

Dashboard features

  • Connected clients — See all currently connected SSE sessions with IP address and workspace.
  • Usage statistics — Token counts per workspace, per model, over time.
  • Data management — Export all data as a ZIP, or purge conversations older than N days.
  • Server settings — Change port, toggle verbose logging, set rate limits.
  • Announcement broadcast — Send a system message to all connected clients instantly.
🚨
Change the default password The default credentials admin / admin123 are public knowledge. Anyone on your network who finds the admin URL can log in if you haven't changed them. Do this before sharing your server with others.
